1. 对于MTK平台智能手机，请使用工程模式里的System Logger工具,不要使用USB直接抓，因为那样抓的不全。
操作步骤：刷完机后，一开机，第一件事就是进工程模式，打开System Logger,同时，更新下系统时间，那样LOG才对得上时间。
打开LOG方式：11059，11091，11071，12007使用*#36446337*,12009及以后机型还可以使用*#800#

2. 测试一段时间后，如果没发现问题，请使用"clrLog(一键清理LOG).bat"清除一下，以免log累积太多，影响分析。

3. 发现bug后，首先请截屏（11059，11091，11071，12007截屏长按menu键,12009及以后机型同时按下电源键与音量下键<或上键>），
   截屏的目的，一是让开发可以对BUG有一个直观的印象，二是可以确认出问题的准确时间。
   然后，请使用"getLog(一键导出LOG).bat"从手机中导出log,导完确认无误后，再使用clrlog.bat清除一下，以免这次的LOG给下次造成影响。
   导出LOG默认存储在d:\log目录下，文件名为log_pack_YY_MM_DD_HH_MM_SS
  
4. 测试关机闹钟抓LOG，需要使用串口，使用USB或工程模式抓的LOG会不全。
   1）串口LOG抓取的话，首先要准备一根USB转串口的线，这个可以从实验室借。
   2）然后安装驱动：串口LOG\CDM20802_Setup.exe
   3）抓取方式，请参考：串口LOG\Use HyperTernial to Capture Log.docx

5.  11091和11071在开机后，adb不是一直连接的，因此不方便抓取或者查看动态log。鉴于此，在版本中添加了一个adb开关，
    可使adb在开机过程中保持连接，且在选择“仅充电”和“存储装置”后adb也一直连接。操作步骤如下：
    1) 连上手机，选择“串行接口/PC套件"
	2) 电脑上运行CMD，输入adb shell
    2) 设置系统属性值“persist.oppo.adb.alwayson”为“yes”，即可使adb保持一直连接（重启手机仍然有效）；设置成no即可还原。
	   设置方法，输入：setprop persist.oppo.adb.alwayson  yes
6：
    12009最新基线工程模式下的SystemLog设置界面(*#800#)，集成了一个oppoMobilelog的apk，主要有两个功能：
	1.可以把mainlog保存到/data目录下。适用于选择“存储装置”卸载sd卡时log丢失的情况（之前59和91是可以把log保存到内存的，但是后来mtk取消了这个功能）。
	2.异常情况下连接adb。适用于手机出现异常时，插入usb没有弹出框，设置界面的“USB调试”不能开启等无法连接adb的情况。
	此功能有异常请联系 周炳书
   
7.
 前在产线或者部分同事有遇到这样的情况：刷完机后第一次开机概率性出现异常，而这时候mtkMobileLog是关闭的，
 以致于在不连接ddms的情况下无法抓取到log进行分析。鉴于此，在12009最新的构建版本里已实现了自动保存刷机完后第一次开机log的功能。具体如下：
    1.刷完机第一次开机的mainlog会自动保存在/data/bootlog目录下。且默认情况下，该目录保存的log也一定是刷完机后第一次开机的log。
    之后再开关机，不会再触发此项功能。
    2.log记录的时间段：从刷完机第一次开机――》接收到ACTION_BOOT_COMPLETED广播之后的一分钟（即开机完成之后的一分钟）。
   注：目前该项功能暂不支持kernel log的抓取；
